{% trans "Common GDPR Mistakes & How to Avoid Them" %}

❌ {% trans "What NOT to do:" %}
  • {% trans "Pre-ticked consent checkboxes" %}
  • {% trans "Bundling consent with terms of service" %}
  • {% trans "Vague consent language ('we may use your data')" %}
  • {% trans "Forcing consent as condition for service (unless truly necessary)" %}
✅ {% trans "How to fix:" %}
  • {% trans "Unchecked boxes by default" %}
  • {% trans "Separate consent for each processing purpose" %}
  • {% trans "Clear, specific language: 'I agree to receive marketing emails about products'" %}
  • {% trans "Record: timestamp, IP, consent text version, user agent" %}
  • {% trans "Easy withdrawal mechanism (one-click unsubscribe)" %}

❌ {% trans "What NOT to do:" %}
  • {% trans "No documentation of processing activities" %}
  • {% trans "Incomplete records (missing legal basis, retention periods)" %}
  • {% trans "Outdated documentation" %}
✅ {% trans "How to fix:" %}
  • {% trans "Use 'Processing Activities' module to document ALL processing" %}
  • {% trans "Include: purpose, data categories, subjects, legal basis, recipients, retention, security" %}
  • {% trans "Review and update records quarterly" %}
  • {% trans "Ready to produce on demand for DPA audit" %}

❌ {% trans "What NOT to do:" %}
  • {% trans "Ignoring or delaying DSR responses beyond 30 days" %}
  • {% trans "Charging fees for basic access requests" %}
  • {% trans "Providing incomplete data exports" %}
  • {% trans "Refusing erasure without valid legal grounds" %}
✅ {% trans "How to fix:" %}
  • {% trans "Respond within 30 days (extend by 60 with justification)" %}
  • {% trans "Free of charge for first request" %}
  • {% trans "Export ALL data in machine-readable format (JSON, CSV)" %}
  • {% trans "Only refuse erasure if: legal obligation, public interest, legal defense" %}
  • {% trans "Use DSR module to track deadlines and audit trail" %}

❌ {% trans "What NOT to do:" %}
  • {% trans "Not reporting breach to DPA within 72 hours" %}
  • {% trans "Hiding breach from affected individuals" %}
  • {% trans "No incident documentation" %}
  • {% trans "No lessons learned or mitigation" %}
✅ {% trans "How to fix:" %}
  • {% trans "Immediate containment and assessment" %}
  • {% trans "Log in Breach Management module within hours" %}
  • {% trans "Notify DPA within 72h if breach 'likely to result in risk'" %}
  • {% trans "Notify individuals 'without undue delay' if high risk to rights" %}
  • {% trans "Document: nature, categories, approx. number, consequences, measures taken" %}
  • {% trans "Conduct post-incident review and update security measures" %}

❌ {% trans "What NOT to do:" %}
  • {% trans "Keeping data 'just in case' indefinitely" %}
  • {% trans "No defined retention periods" %}
  • {% trans "Not deleting data when purpose is fulfilled" %}
✅ {% trans "How to fix:" %}
  • {% trans "Define retention policies for each data category" %}
  • {% trans "Base retention on: legal requirements, business need, data minimization" %}
  • {% trans "Automate deletion where possible (scheduled jobs)" %}
  • {% trans "Document deletion method and responsibility" %}
  • {% trans "Review retention policies annually" %}

                                {% trans "DSR (Data Subject Request)" %}
                                {% trans "Rights" %}
                            

                            

                                
{% trans "Request by an individual to exercise their rights under GDPR (access, rectification, erasure, restriction, portability, objection)." %}

                                {% trans "Deadline:" %} {% trans "30 days (+ 60 days extension with justification)" %}
                            

                        


                        
                        

                            

                                {% trans "Lawful Basis" %}
                                {% trans "Legal" %}
                            

                            

                                
{% trans "One of six legal grounds required to process personal data lawfully (Art. 6): consent, contract, legal obligation, vital interests, public task, legitimate interests." %}

                                {% trans "Most common:" %} {% trans "Consent (marketing), Contract (service delivery), Legal obligation (accounting)" %}

                                {% trans "Definition" %}
                            

                            

                                
{% trans "Any information relating to an identified or identifiable natural person (Art. 4(1))." %}

                                {% trans "Examples:" %} {% trans "Name, email, IP address, cookie ID, location data, online identifier" %}
                            

                        


                        

                            

                                {% trans "Data Processor" %}
                                {% trans "Role" %}
                            

                            

                                
{% trans "Entity that processes personal data on behalf of the controller (Art. 4(8))." %}

                                {% trans "Example:" %} {% trans "Cloud hosting provider, email service provider, payroll company" %}
                            

                        


                        

                            

                                {% trans "Pseudonymization" %}
                                {% trans "Technical" %}

                                {% trans "Example:" %} {% trans "Replacing user IDs with hashed tokens, separating identifying info" %}
